## 测试前置条件

| 项目 | 要求 |
|------|------|
| 环境 | 后端 + 前端 + PostgreSQL 正常启动 |
| LLM API | 已配置可用的 LLM provider（或设置 `LLM_JUDGE_ENABLED=false` 走降级） |
| 测试账号 | admin / reviewer / auditor 各一个 |
| 测试文件 | 至少 3 家投标人的压缩包（含 .docx + .xlsx），其中至少 1 份加密 |

## 1. 用户认证与权限

### AT-1.1 正常登录
| 项 | 内容 |
|-----|------|
| **前置** | 已有激活账号 |
| **步骤** | 1. 打开登录页 → 输入正确用户名/密码 → 点击登录 |
| **预期** | 跳转到项目列表页；页面右上角显示当前用户名 |

### AT-1.2 密码错误锁定
| 项 | 内容 |
|-----|------|
| **步骤** | 1. 连续输入错误密码 5 次 |
| **预期** | 第 5 次后返回 429，提示账号已锁定；15 分钟内无法登录 |

| 项 | 内容 |
|-----|------|
| **前置** | 管理员已将该账号设为 is_active=false |
| **步骤** | 1. 使用被禁用账号登录 |
| **预期** | 返回 403，提示账号已禁用 |

### AT-1.4 首次登录强制改密
| 项 | 内容 |
|-----|------|
| **前置** | 管理员新建用户（must_change_password=true） |
| **步骤** | 1. 使用新账号登录 |
| **预期** | 登录后自动跳转到修改密码页，必须修改密码后才能进入系统 |

### AT-1.5 角色权限隔离
| 项 | 内容 |
|-----|------|
| **步骤** | 1. 以 reviewer 角色登录 → 尝试访问 /admin 页面 |

### AT-1.6 登出

## 6. 检测执行与实时进度

### AT-6.1 启动检测
| 项 | 内容 |
|-----|------|
| **前置** | 项目有 ≥2 个投标人处于终态（priced / identified） |
| **步骤** | 1. 点击"开始检测"按钮 |
| **预期** | 项目状态变为 analyzing；创建 AgentTask 批次（C(n,2)×7 对比任务 + 4 全局任务） |

### AT-6.2 检测前置校验
| 项 | 内容 |
|-----|------|
| **前置** | 项目只有 1 个投标人 |
| **步骤** | 1. 尝试点击"开始检测" |
| **预期** | 按钮禁用或提示：投标人数量不足 |

### AT-6.3 检测进度实时推送
| 项 | 内容 |
|-----|------|
| **步骤** | 1. 启动检测后停留在页面 |
| **预期** | SSE 推送 agent 进度：显示进度条 + 当前运行/成功/失败/超时数量；检测完成时收到 report_ready 事件 |

### AT-6.4 检测状态查询
| 项 | 内容 |
|-----|------|
| **步骤** | 1. 检测过程中调用 GET /analysis/status |
| **预期** | 返回当前检测版本号 + 所有 AgentTask 的状态快照 |

### AT-6.5 单个 Agent 失败不影响整体
| 项 | 内容 |
|-----|------|
| **步骤** | 1. 模拟某个 Agent 执行失败（如 LLM 超时） |
| **预期** | 该 Agent 标记为 failed；其他 Agent 继续执行；最终报告仍然生成（失败维度计 0 分） |

## 7. 检测报告与评分

### AT-7.1 报告总览
| 项 | 内容 |
|-----|------|
| **前置** | 检测已完成 |
| **步骤** | 1. 进入报告页面 |
| **预期** | 显示：总分（0–100）、风险等级标签（high/medium/low）、LLM 综合结论、11 个维度卡片 |

### AT-7.2 维度排序展示
| 项 | 内容 |
|-----|------|
| **步骤** | 1. 查看维度列表排序 |
| **预期** | 按铁证优先（is_ironclad desc）→ 分数从高到低（best_score desc）排列 |

### AT-7.3 11 维度覆盖
| 项 | 内容 |
|-----|------|
| **步骤** | 1. 检查报告中的维度列表 |
| **预期** | 包含全部 11 个维度：text_similarity / section_similarity / structure_similarity / metadata_author / metadata_time / metadata_machine / price_consistency / price_anomaly / error_consistency / image_reuse / style |

### AT-7.4 风险等级阈值
| 项 | 内容 |
|-----|------|
| **步骤** | 1. 验证不同总分对应的风险等级 |
| **预期** | ≥70 → high（红色），40–69 → medium（黄色），<40 → low（绿色） |

### AT-7.5 铁证触发机制
| 项 | 内容 |
|-----|------|
| **前置** | 检测发现铁证（如元数据作者完全相同） |
| **步骤** | 1. 查看报告中铁证维度 |
| **预期** | 铁证维度标记 is_ironclad；总分不低于 85 分；风险等级强制为 high |

### AT-7.6 LLM 综合研判
| 项 | 内容 |
|-----|------|
| **前置** | LLM_JUDGE_ENABLED=true 且 LLM 可用 |
| **步骤** | 1. 检测完成后查看报告结论 |
| **预期** | 显示 LLM 生成的综合研判文字；LLM 只能升分不能降分 |

### AT-7.7 LLM 不可用降级
| 项 | 内容 |
|-----|------|
| **前置** | LLM_JUDGE_ENABLED=false 或 LLM 不可用 |
| **步骤** | 1. 检测完成后查看报告结论 |
| **预期** | 显示降级模板结论（"AI 综合研判暂不可用…"）；总分 = 纯公式得分 |

### AT-7.8 维度详情与证据
| 项 | 内容 |
|-----|------|
| **步骤** | 1. 点击某个维度卡片查看详情 |
| **预期** | 显示：最高分、铁证标记、证据摘要、Agent 状态统计（成功/失败/超时/跳过） |
